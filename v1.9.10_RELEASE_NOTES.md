## üéØ What's New in v1.9.10

### Repository Consolidation & Cleanup üßπ

**Script Consolidation:**

- Removed empty `CLEANUP_DOCS.ps1`
- Deprecated 4 redundant scripts with clear migration paths:
  - `CLEANUP_OBSOLETE_FILES.ps1` ‚Üí Use `CLEANUP_COMPREHENSIVE.ps1` (canonical)
  - `CREATE_PACKAGE.ps1` ‚Üí Use `CREATE_DEPLOYMENT_PACKAGE.ps1` (more comprehensive)
  - `run_in_venv_clean.py` ‚Üí Use `run_in_venv.py` (actively used in pre-commit)
  - All deprecated scripts show warnings directing to canonical alternatives

**Installer Consolidation (PowerShell ‚Üí Inno Setup):**

- Deprecated 4 PowerShell-based installer wizards in `tools/installer/`
- Canonical installer: `installer/SMS_Installer.iss` (Inno Setup)
- Benefits: Better Windows integration, code signing support, simpler distribution
- All legacy scripts retained with deprecation notices for backward compatibility

**Documentation Improvements:**

- Fixed broken references in `START_HERE.md`
- Consolidated monitoring docs: `MONITORING_ARCHITECTURE.md` ‚Üí canonical `MONITORING.md`
- Consolidated development guides: 3 guides ‚Üí canonical `DEVELOPER_GUIDE_COMPLETE.md`
  - `DEVELOPMENT.md`, `DEVELOPER_FAST_START.md`, `DEVELOPMENT_SETUP_GUIDE.md` marked as reference
- Updated `docs/DOCUMENTATION_INDEX.md` to clarify canonical vs reference materials
- Enhanced `scripts/README.md` with comprehensive consolidation documentation

### Security Enhancements üîí

**Environment File Protection:**

- Removed `.env.qnap` from git tracking (contained passwords and secret keys)
- Added explicit `.gitignore` entry to prevent future tracking
- File remains locally for users but won't be committed to repository

**Binary File Management:**

- Added `.gitignore` rules for `*.exe` files to prevent repository bloat
- Exception for `installer/SMS_Installer_*.exe` (tagged release installers only)
- Prevents large binaries (~5MB each) from inflating git history
- Installers distributed via GitHub Releases instead of git
