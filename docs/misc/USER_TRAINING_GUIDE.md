## Getting Started

### What is SMS Student Management System?

SMS is a complete student management platform that helps manage:
- ✅ Student records and information
- ✅ Course enrollment and management
- ✅ Grade recording and tracking
- ✅ Attendance tracking
- ✅ Data import/export

### Who Should Use This?

- **Administrators:** Manage users, courses, and reports
- **Teachers:** Track students, manage courses, enter grades
- **Students:** View grades, enrollment, schedules

### What You'll Need

- Any computer (Windows, Mac, or Linux)
- Internet browser (Chrome, Firefox, Edge, Safari)
- Username and password (provided by your administrator)
- About 100 MB of storage

## Installation

### Download the Application

**Step 1: Get the Executable**
```
Download file: SMS_Native_Lite_Simple.exe (67 MB)
Source: Your IT administrator or company download page
Save location: Desktop or Documents (your choice)
```

**Step 2: Place It Somewhere Safe**
```
Recommended locations:
  - Desktop (easy access)
  - Documents folder (organized)
  - Program Files (permanent)
  - Network drive (shared access)
```

### Run the Application

**Step 3: Start SMS Lite**
```
Option A - Windows:
  1. Double-click SMS_Native_Lite_Simple.exe
  2. A console window will appear
  3. Wait 8-10 seconds (server starting)
  4. Your browser will open automatically
  5. You'll see the SMS login page

Option B - Linux:
  1. Open terminal
  2. Navigate to folder with exe
  3. Run: ./SMS_Native_Lite_Simple.exe
  4. Wait 8-10 seconds
  5. Browser opens to http://127.0.0.1:8000
```

### System Requirements Check

```
✓ Windows 10/11 or Linux (Ubuntu/Debian/CentOS)
✓ At least 500 MB free storage
✓ At least 512 MB RAM available
✓ Port 8000 must be available (not blocked)
✓ Modern web browser installed
```

If you have issues, see [Troubleshooting](#troubleshooting) section.

## FAQ - Frequently Asked Questions

### Q: The application won't start

**A:**
```
Try these steps:
  1. Make sure you have Windows 10/11 or Linux
  2. Check that port 8000 is not being used
  3. Try restarting your computer
  4. Check that you have 512 MB RAM available
  5. Delete and re-download the executable
  6. Contact IT support if issue persists
```

### Q: Browser doesn't open automatically

**A:**
```
Workaround:
  1. Run SMS_Native_Lite_Simple.exe
  2. Open your web browser (Chrome, Firefox, etc.)
  3. In address bar, type: http://127.0.0.1:8000
  4. Press Enter
  5. Login page appears
  Note: Server is running fine, just needs manual browser open
```

### Q: I forgot my password

**A:**
```
Contact your administrator:
  - They can reset your password
  - Or create a new account for you
  - You'll get temporary credentials
  - Change password on next login
```

### Q: Can I run multiple instances?

**A:**
```
No, because:
  - Only one application can use port 8000
  - Database would conflict
  - Not designed for multiple instances

Solution:
  - Run on different computers
  - Run on different ports (requires configuration)
  - Use network version (if available)
```

### Q: What if I lose my data?

**A:**
```
Backup your data:
  1. Export regularly (📂 Import/Export)
  2. Save to USB drive
  3. Upload to cloud storage
  4. Notify IT about backup needs

Restore from backup:
  1. Re-import your exported file
  2. Or ask IT to restore from their backup
  3. Database file is at: %LOCALAPPDATA%\SMS_Native_Lite_Simple\
```

### Q: Can I access from home/remotely?

**A:**
```
Default: NO (local network only)

To enable remote access:
  1. Contact IT department
  2. They can setup VPN or reverse proxy
  3. This adds security layer
  4. Then you can access from anywhere

Without IT setup:
  - Only works on office network
  - Only works on same machine
```

### Q: Is my data secure?

**A:**
```
Security features:
  ✓ Login with password
  ✓ Role-based access (what you can see)
  ✓ Database encryption (optional)
  ✓ Audit logs (who did what)

Your responsibility:
  ✓ Keep password confidential
  ✓ Don't share login credentials
  ✓ Lock computer when away
  ✓ Change default password
```

### Q: How do I update the application?

**A:**
```
Method 1 - Automatic (if IT configured):
  You'll see update notification
  Click "Update"
  New version installs

Method 2 - Manual:
  Contact IT for new version
  Delete old exe
  Download new exe
  Run it

Method 3 - Network share:
  IT updates central copy
  You automatically get new version
  No action needed
```

### Q: Can I use this offline?

**A:**
```
Default SQLite database:
  ✓ Yes, fully offline
  ✓ No internet needed
  ✓ All data stored locally
  ✓ Works anywhere

If using QNAP PostgreSQL:
  ✗ No, requires network
  ✓ Can work if network available
  ✗ Cannot access if QNAP is down
```

### Q: What's the difference between roles?

```
Administrator:

Teacher:

  ✓ Can track attendance
```
