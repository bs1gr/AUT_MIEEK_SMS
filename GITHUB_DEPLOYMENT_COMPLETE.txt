## ğŸ†˜ Common Questions

**Q: How do I run tests locally?**
```powershell
.\COMMIT_READY.ps1 -Quick   # Quick validation
.\COMMIT_READY.ps1 -Full    # Full validation including all tests
```

**Q: What happens if tests fail?**
- Pipeline stops, PR shows failed status
- Review error logs in GitHub Actions UI
- Fix locally and push again (force is OK for branches)

**Q: How do I deploy to production?**
```powershell
# Tag release
git tag -a v2.0.0 -m "Release v2.0.0"
git push origin v2.0.0

# Pipeline automatically creates release and publishes
# Then deploy with:
docker pull ghcr.io/yourorg/student-management-system:v2.0.0-build.latest
.\DOCKER.ps1 -Start
```

**Q: Can I use different Python/Node versions?**
Edit `.github/workflows/ci-cd-pipeline.yml` and update:
```yaml
python-version: ['3.10', '3.11', '3.12']  # Test matrix
node-version: ['18.x', '20.x']
```

**Q: How do I skip the pipeline?**
Don't! It ensures quality. But if needed, tag commit with `[skip ci]`:
```
git commit -m "docs: update readme [skip ci]"
```
